# Product Spec

## Linked Issue

GH-136

## 用户问题

Sage 有完整的 memory/learning 实现却从不进入推理路径，每个会话冷启动、无法从历史成功/失败中学习。用户期望一个能记住项目约定、复用过往经验、避免重犯错误的 agent。

## 目标

- 让跨会话记忆与学习模式在会话开始时可用，并在构建 prompt/context 时被注入。
- 记录成功与失败两类轨迹，检索时能返回失败教训（W-37）。
- 注入受容量上界与敏感信息 redaction 约束，且默认可关闭。

## 非目标

- 不实现云端记忆后端。
- 不改变 `SessionNotesTool` 工具接口。
- 不引入无界内存增长。

## Behavior Invariants

1. agent 启动时，全局 memory/learning 存储被初始化到可配置路径；初始化失败时以 error 级暴露且不静默吞掉（U-29）。
2. 构建发送给 LLM 的上下文时，与当前任务相关的记忆与学习模式被注入到 prompt 的专用区段。
3. 注入内容量受上界约束；超出时按相关性截断并记录被丢弃数量，不无界增长。
4. 注入内容经过 redaction，不泄漏 secret/token/凭据/敏感路径。
5. 学习记录带 outcome 标记（成功/失败），失败轨迹在被裁剪前先提炼为可检索的教训（W-37）。
6. 记忆/学习功能可通过配置关闭；关闭时 agent 行为与注入前一致，且不产生错误输出。
7. 相关记忆为空时按空处理（不注入区段），不阻断正常流程。

## 验收标准

- [ ] 启动初始化记忆/学习（或维护者选择：显式声明诊断专用并从能力叙事移除）。
- [ ] prompt/context 注入相关记忆与学习模式，受上界与 redaction 约束。
- [ ] 学习记录含 outcome 标记，检索可返回失败教训。
- [ ] 有开关；关闭时行为回归无注入基线。

## 边界情况

- 存储路径不可写：初始化 error 级报告，agent 仍可运行但明确告知记忆不可用（非静默）。
- 注入后上下文接近窗口上限：与既有 auto_compact 协作，记忆注入不得挤爆窗口。

## 发布说明

新增能力，默认行为需在 PR 中声明（默认开启还是默认关闭）；涉及跨会话数据存储位置需文档化。

## 开放问题

- 记忆相关性检索用什么信号（任务文本 / 文件路径 / 工具序列）？由 tech spec 定。
- 默认开启还是 opt-in？建议 opt-in 起步以控制上下文与幻觉面。
